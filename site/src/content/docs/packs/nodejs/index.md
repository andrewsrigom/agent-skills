## Skills

- [Node.js API Latency And Throughput](/packs/nodejs/nodejs-api-latency-and-throughput/) — Use when a Node.js API is slow because of per-request work, bad concurrency shape, I/O amplification, or weak throughput defaults. Covers p95-first analysis, request path simplification, batching, cancellation, and avoiding fake optimizations that hurt operability.
- [Node.js Best Practices](/packs/nodejs/nodejs-best-practices/) — Use when building or debugging Node.js applications and libraries. Covers modules, async flows, streams, testing, shutdown, environment handling, logging, performance, and modern TypeScript execution choices in Node.js.
